Grant types are a list of protocol flows that the client application may use. This service rejects non-registered protocol flows.

authorization_code

Denotes that the client application may use the authorization code flow. If the value of "response_type" request parameter of an authorization request from your client application may contain "code", check this.

implicit

Denotes that the client application may use the implicit flow. If the value of "response_type" request parameter of an authorization request from your client application may contain "token" or "id_token" (or both), check this.

password

Denotes that the client application may use the resource owner password credentials flow. If your client application may use "password" as the value of "grant_type" request parameter of a token request, check this.

client_credentials

Denotes that the client application may use the client credentials flow. If your client application may use "client_credentials" as the value of "grant_type" request parameter of a token request, check this.

refresh_token

Denotes that the client application wants to be issued a refresh token. If your client application may use "refresh_token" as the value of "grant_type" request parameter of a token request, check this.
