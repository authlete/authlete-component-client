Response types are a list of values of "response_type" request parameter that the client application may specify. An authorization request with a non-registered "response_type" This service rejects non-registered "response_type" values.

In RFC 6749, valid values of "response_type" request parameter are just two, namely, "code" and "token". However, OpenID Connect has extended them to combinations of "code", "id_token" and "token", or "none". As a result, the number of valid values is 8. See OAuth 2.0 Multiple Response Type Encoding Practices for details.

code

Denotes that the client application may request an authorization code. If the client application may use the authorization code flow, check this.

Note that even if the value of "response_type" request parameter is "code", if "scope" request parameter of the authorization request contains "openid", an ID token will be issued at the token endpoint as well as an access token.

token

Denotes that the client application may request an access token at the authorization endpoint. If the client application may use the implicit flow, check this.

id_token

Denotes that the client application may request an ID token at the authorization endpoint.

code token

Denotes that the client application may request an authorization code and an access token at the same time at the authorization endpoint.

code id_token

Denotes that the client application may request an authorization code and an ID token at the same time at the authorization endpoint.

id_token token

Denotes that the client application may request an ID token and an access token at the same time at the authorization endpoint.

code id_token token

Denotes that the client application may request an authorization code, an ID token and an access token at the same time at the authorization endpoint.

none

Denotes that the client application may send an authorization request which does not expect any object to be returned. This use case is special. See 4. None Response Type of OAuth 2.0 Multiple Response Type Encoding Practices for details.
