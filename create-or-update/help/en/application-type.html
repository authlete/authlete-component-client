Application type denotes the basic category of a client application. Either "web" or "native" can be chosen. Or, you can decide not to choose either.

The value of this configuration parameter affects the validation steps for a redirect URI. When "web" is chosen, the scheme of redirect URIs for the implicit flow must be "https" and the hostname part of them must not be "localhost". On the other hand, when "native" is chosen, the scheme of redirect URIs can be a custom scheme, or "http" only when the hostname part is "localhost".

This configuration parameter corresponds to "application_type" defined in 2. Client Metadata of OpenID Connect Dynamic Client Registration 1.0.
