認可種別はクライアントアプリケーションが使用する可能性のあるプロトコルフローのリストです。 登録されていないプロトコルフローはサービスにより拒否されます。

authorization_code

クライアントアプリケーションがauthorization code フローを使用する可能性があることを示します。 クライアントアプリケーションからの authorization リクエストの response_type リクエストパラメーターが code を含むことがある場合はチェックしてください。

implicit

クライアントアプリケーションがimplicit コードフローを使用する可能性があることを示します。 クライアントアプリケーションからの authorization リクエストの response_type リクエストパラメーターが token もしくは id_token (または両方) を含むことがある場合はチェックしてください。

password

クライアントアプリケーションがresource owner password credentials フローを使用する可能性があることを示します。 クライアントアプリケーションが token リクエストの grant_type リクエストパラメーターの値として password を使用する可能性がある場合はチェックしてください。

client_credentials

クライアントアプリケーションがclient credentials フローを使用する可能性があることを示します。 クライアントアプリケーションが token リクエストの grant_type リクエストパラメーターの値として client_credentials を使用する可能性がある場合はチェックしてください。

refresh_token

クライアントアプリケーションがリフレッシュトークンの発行を希望することを示します。 クライアントアプリケーションが token リクエストの grant_type リクエストパラメーターの値として refresh_token を使用する可能性がある場合はチェックしてください。
