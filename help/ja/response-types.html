応答種別は、クライアントアプリケーションが response_type リクエストパラメータの値として指定する可能性のある値のリストです。 登録されていない値を response_type に指定した authorization リクエストは拒否されます。

RFC 6749 では、response_type リクエストパラメーターの有効な値は code と token の二つだけでした。 しかし、OpenID Connect はこれを、code, id_token, token の任意の組み合わせ、および none へと拡張しました。 この結果、有効な値の数は 8 となりました。 詳細は OAuth 2.0 Multiple Response Type Encoding Practices を参照してください。

code

クライアントアプリケーションが authorization code をリクエストする可能性があることを示します。 クライアントアプリケーションが authorization code フローを使用する可能性がある場合はチェックしてください。

注意: response_type リクエストパラメーターの値が code であったとしても、その authorization リクエストの scope パラメーターに openid が含まれる場合、token エンドポイントにおいて、アクセストークンに加えて ID トークンも発行されます。

token

クライアントアプリケーションが authorization エンドポイントでアクセストークンをリクエストする可能性があることを示します。 クライアントアプリケーションが implicit フローを使用する可能性がある場合はチェックしてください。

id_token

クライアントアプリケーションが authorization エンドポイントで ID トークンをリクエストする可能性があることを示します。

code token

クライアントアプリケーションが authorization エンドポイントで authorization code とアクセストークンをリクエストする可能性があることを示します。

code id_token

クライアントアプリケーションが authorization エンドポイントで authorization code と ID トークンをリクエストする可能性があることを示します。

id_token token

クライアントアプリケーションが authorization エンドポイントで ID トークンとアクセストークンをリクエストする可能性があることを示します。

code id_token token

クライアントアプリケーションが authorization エンドポイントで authorization code、ID トークン、アクセストークンをリクエストする可能性があることを示します。

none

クライアントアプリケーションが、オブジェクトの返却を求めない authorization リクエストを投げる可能性があることを示します。このユースケースは特別です。 詳細は OAuth 2.0 Multiple Response Type Encoding Practices の 4. None Response Type を参照してください。
